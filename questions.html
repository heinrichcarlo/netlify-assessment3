Detailed responses to the Netlify interview questions

How did you build this site and what challenges did you face?

I built this site with HTML/CSS/JavaScript for optimal compatibility with Netlify's static hosting. The authentication uses Firebase for its security and scalability with static sites. The UI features Poppins fonts and Font Awesome icons for a modern, accessible design.

Key Challenge: Initially, Firebase functions (registerUser, loginUser) were undefined due to script loading issues. I resolved this by:

  • Ensuring Firebase SDKs load before my auth.js
  • Explicitly attaching functions to window.authFunctions
  • Adding error boundaries and console logs for debugging

DNS challenges for non-technical users

Challenge 1: Nameserver Configuration

Many users struggle to update nameservers with their registrar (e.g., GoDaddy, Namecheap) and in some cases customers don't know where the domain is registered and who holds the current NS. Without this step, DNS changes won't propagate. A visual guide or automated domain connection (like Netlify's current flow) significantly helps.

Challenge 2: SSL/TLS Setup

Users often confuse DNS records (A, CNAME) with SSL certificates. While Netlify's automatic HTTPS is excellent, users may not understand why mixed content warnings occur when they hardcode http:// assets in their sites.

Troubleshooting "site won't build" error

Investigation Steps:

  1. Check build logs for the exact error preceding exit code 2
  2. Verify the build command in Netlify UI matches the repo's requirements
  3. Look for dependency conflicts (Node.js version mismatches)
  4. Check for missing files in required directories (e.g., package.json)

Sample Customer Response:

Function deployment experience

I deployed a simple "Hello World" function:

1. Created netlify/functions/hello.js:

exports.handler = async () => {
  return { 
    statusCode: 200, 
    body: "Hello World!" 
  };
};

2. Configured in netlify.toml:

[functions]
  directory = "netlify/functions"

Challenge: Initially received 404 errors because the functions directory was incorrectly specified.

Solution: Verified the directory path matched Netlify's requirements and tested locally using Netlify Dev.

Responding to security reports

Immediate Response:

"Thank you for reporting this. We take security very seriously and have initiated an urgent investigation. We'll provide updates as soon as possible."

Investigation Steps:

  1. Attempt to reproduce in staging environment
  2. Review server logs and monitoring systems
  3. Isolate potentially affected systems

Escalation Path:

  • Security Team → CTO → PR/Comms (if public disclosure needed)
  • Document all steps for post-mortem analysis
